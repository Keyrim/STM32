Répartitions des adresses
	0-99 	=> pour le drone 
		0-19	=> 1 octet 
		20-39	=> 2 octets 
		40-59	=> 3 octets 
		60-79	=> 4 octets 
		80-99	=> 5 octets 
	100-199 => pour le pc  
		100-119	=> 1 octet 
		120-139	=> 2 octets 
		140-159	=> 3 octets 
		160-179	=> 4 octets 
		180-199	=> 5 octets 
	200-255	=> pour la ground station
		200-209	=> 1 octet 
		210-219	=> 2 octets 
		220-229	=> 3 octets 
		230-239	=> 4 octets 
		240-249	=> 5 octets 

Le nombre d'octect n'inclut pas l'octect envoyé, en effet on dois toujours avoir un un id plus un msg, mais un msg d'erreur par exemple doit être composé d'un id d'une donné 
On supprime au passage les ids inutile (moteur et telecomande un par un etc)
